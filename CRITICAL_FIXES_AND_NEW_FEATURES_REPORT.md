```python
# ПРАВИЛЬНО (новая логика):
def calculate_points_transaction(amount_gross, accrual_pct, redeem_rate, points_to_spend):
    if points_to_spend > 0:
        # Если пользователь ТРАТИТ баллы - начисления НЕТ
        return {
            'points_spent': points_to_spend,
            'points_earned': 0,  # КРИТИЧЕСКОЕ ИСПРАВЛЕНИЕ!
            'net_change': -points_to_spend,
            'operation_type': 'spending_only'
        }
    else:
        # Если пользователь НЕ тратит баллы - есть начисление
        if amount_gross >= MIN_PURCHASE_FOR_POINTS:
            points_earned = math.floor((amount_gross * accrual_pct / 100) / redeem_rate)
        else:
            points_earned = 0
        return {
            'points_spent': 0,
            'points_earned': points_earned,
            'net_change': points_earned,
            'operation_type': 'earning_only'
        }
```

### 🎯 **ПРИНЦИП: ЛИБО ТРАТИШЬ, ЛИБО НАКАПЛИВАЕШЬ**
- ✅ При списании баллов (points_to_spend > 0) → `points_earned = 0`
- ✅ При накоплении (points_to_spend = 0) → `points_earned = расчет по формуле`
- ✅ Никогда не делать и списание, и начисление одновременно
- ✅ Обновлены все интерфейсы с объяснением логики пользователю

```markdown
## ⚠️ ВАЖНОЕ ПРАВИЛО: ЛИБО ТРАТИШЬ, ЛИБО НАКАПЛИВАЕШЬ

При каждой покупке вы выбираете одну из стратегий:

### 🎁 Стратегия "Накопление"
- Не тратите баллы
- Получаете только базовую скидку партнера
- ПОЛУЧАЕТЕ начисление новых баллов за покупку

### 💎 Стратегия "Трата"  
- Тратите накопленные баллы на дополнительную скидку
- Получаете базовую + дополнительную скидку
- НЕ ПОЛУЧАЕТЕ начисление за эту покупку

### Пример:
Покупка: 200,000₫ в кафе (5% начисление, 10% базовая скидка)

Стратегия "Накопление":
• Скидка: 20,000₫ (10%)
• К доплате: 180,000₫  
• Получите: +2 балла

Стратегия "Трата" (100 баллов):
• Базовая скидка: 20,000₫ (10%)
• Скидка за баллы: 500,000₫
• К доплате: 180,000₫ - 500,000₫ = -320,000₫ (переплата невозможна)
• Фактическая скидка: ограничена 50% = 100,000₫
• К доплате: 100,000₫
• Спишется: 20 баллов (100,000₫ ÷ 5,000₫)
• Получите: 0 баллов (тратили накопления)
```
